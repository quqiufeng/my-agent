# WeChat OCR 模块技术文档

本文档描述「微信机器人消息入口」的底层实现：如何定位桌面「小龙虾」微信窗口、识别聊天文字、并把消息内容交给上层 Lua 脚本处理。

---

## 一、整体定位

```

      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│ 桌面「小龙虾」微信客户端                                      │
└─────────────────────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│  Lua 逻辑层 (wechat_ocr/init.lua)                           │
```

## 二、微信窗口结构识别

### 2.1 横向三区域结构

微信桌面版窗口横向划分为三个竖状功能区域：

```
  ┌──────┬─────────────────────┬──────────────────────────┐
  │ 第一 │ 第二                │ 第三                     │
  │      │                     │                          │
  │ 图标 │  列表显示区域        │  内容展示区              │
  │ 功能 │                     │  (当前选中项的展示面板)  │
  │ 区域 │  显示:               │                          │
  │      │  · 聊天列表         │  标题/名称栏             │
  │      │  · 公众号           │  ──────────────────     │
  │      │  · 服务号           │                          │
  │      │  · 微信支付         │  聊天消息 / 内容         │
  │      │  · 文件传输助手     │  (消息/文章/账单等)      │
  │      │  · 群聊 / 单聊      │                          │
  │      │  · 收藏等           │                          │
  │      │                     │  ──────────────────     │
  │      │  选中项有灰色高亮块  │  输入框                  │
  └──────┴─────────────────────┴──────────────────────────┘
```

### 2.2 各区域特征

#### 第一区域：图标功能区

| 属性 | 说明 |
|------|------|
| 位置 | 最左侧竖条 |
| 宽度 | 约 0-2% 的白面板宽度 (~30px) |
| 特征 | 导航图标，选中为绿色，未选中为灰色 |
| 用途 | 切换功能模块（聊天、通讯录、收藏等） |

#### 第二区域：列表显示区域

| 属性 | 说明 |
|------|------|
| 位置 | 第一区域右侧 |
| 宽度 | 约 10-28% 的白面板宽度 (~150-420px) |
| 背景 | 深色/灰色背景 |
| 特征 | 选中项有灰色高亮块 |
| 用途 | 显示功能列表和聊天列表 |
| 内容 | 公众号、服务号、文件传输助手、微信支付、群聊、单聊等 |

#### 第三区域：内容展示区

| 属性 | 说明 |
|------|------|
| 位置 | 最右侧 |
| 宽度 | 约 28-100% 的白面板宽度 |
| 背景 | 白色/浅灰色 |
| 用途 | 显示第二区域选中项的详细内容 |
| 内容 | 聊天消息 / 公众号文章 / 支付记录等 |
| 内部结构 | 标题栏 (0-35px)、内容区 (35~h-180px)、输入框 (h-180px~) |

### 2.3 纵向结构（第三区域内部）

当第三区域显示聊天内容时，其内部纵向结构为：

```
┌──────────────────────────────────────┐
│ 标题栏 (联系人名称 + 功能按钮)  0-35px │
├──────────────────────────────────────┤
│                                      │
│ 聊天消息区域                          │
│ · 时间戳 (居中或靠左)                 │
│ · 对方消息 (靠左)                     │
│ · 我的消息 (靠右)                     │
│ · 系统消息 (居中)                     │
│                                      │
│ 35px ~ h-180px                       │
│                                      │
├──────────────────────────────────────┤
│ 输入框 (文本输入 + 发送按钮)          │
│ h-180px ~ h                          │
└──────────────────────────────────────┘
```

## 三、OCR 识别流程

### 3.1 完整处理步骤

```
Step 1: 捕获策略（三路径）
  ├─ 路径A: xdotool 窗口几何（主）
  │   └─ find_wechat_window() → capture_screen() 直接截取
  │
  ├─ 路径B: 白面板检测（备）
  │   └─ capture_full_screen() → find_white_window() → 截取
  │
  └─ 路径C: 跨桌面切换（兜底）
      └─ find_wechat_xid() → xdotool 将窗口切换到当前桌面
      └─ 等待 200ms → 走路径A 继续

Step 2: 裁剪第三区域（内容展示区）
  └─ 对全窗口做 OCR，识别 HH:MM 时间戳
  └─ 取第一个时间戳右边缘 + 25px 作为第二/三分隔线
  └─ 小屏（< 2000px）减 10px 修正
  └─ 过滤后只保留第三列文字框

Step 3: OCR 推理
  └─ Text Detection (PP-OCRv4 det model)
     └─ 输入: 裁剪后的图像 (3-channel, normalized)
     └─ 输出: 概率图 → 阈值 → 轮廓 → 文本框
  └─ Text Recognition (PP-OCRv4 rec model)
     └─ 输入: 每个检测到的文本框 (48px高, 动态宽)
     └─ 输出: CTC解码 → 中文字符串

Step 4: 返回结果
  └─ JSON 格式:
     {
       "win": {"x": 绝对屏幕X, "y": 绝对屏幕Y,
               "w": 裁剪区域宽, "h": 裁剪区域高},
       "boxes": [
         {"x": 相对坐标, "y": 相对坐标,
          "w": 文本框宽, "h": 文本框高,
          "text": "识别的文字"},
         ...
       ]
     }

Step 5: Lua 层处理
  └─ capture() 直接返回文字列表 (按Y排序)
  └─ monitor() 持续轮询 + 差分检测新消息
  └─ 上层脚本将文本解析为指令并执行
```

### 3.2 输入框上方功能键

第三区域（内容展示区）输入框上方有一排功能图标：

```
索引 1     2     3       4     5       6       7
   😊    📎    🎤    📺    📹      📹      ...
  表情   文件  语音   截图   视频    视频   更多
                                (倒数第二个) (最右边)
```

- **第 3 个图标 `📎`** — 发送文件
- **第 4 个图标 `📺`** — 截图

### 3.3 第三区域底部结构（工具区）

```
┌──────────────────────────────────┐
│ 聊天消息区域                      │
├──────────────────────────────────│← 水平分割线（唯一一条）
│ 😊 📎 🎤 📺 ...  工具栏灰色图标   │
│ 输入框                            │
└──────────────────────────────────┘
```

布局特点：
- 唯一一条水平线将「聊天消息区」和「工具区」隔开
- 线下方直接是图标工具栏 + 输入框，二者之间没有线
- 图标为灰色 (RGB ~180-200)，背景浅灰 (RGB ~237)，对比度低
- 定位方法：Canny + Hough 检测底部最后一条最长水平线

### 3.4 窗口定位方法对比

| 方法 | 原理 | 可靠性 | 场景 |
|------|------|--------|------|
| **xdotool 几何** (主) | 通过窗口标题/类名获取位置 → 直接截图 | ★★★★★ | 常规运行 |
| **白面板检测** (备) | Canny边缘检测 → 最大矩形轮廓 → 内部亮度验证 | ★★★★☆ | xdotool 失败 |
| **跨桌面切换** (兜底) | xdotool 将微信切到当前桌面 → 重新定位 | ★★★★☆ | 微信在其他虚拟桌面 |

## 四、关键算法说明

### 4.1 白面板检测算法

```cpp
WindowRect find_white_window(const cv::Mat &full_screen) {
    1. 灰度化 + Canny边缘检测 (阈值30, 100)
    2. 形态学膨胀连接边缘
    3. 查找外轮廓
    4. 多边形逼近 (过滤非矩形轮廓)
    5. 过滤条件:
       - 面积 > 屏幕面积 5%
       - 宽高比 0.5~3.0
       - 内部平均亮度 > 180
       - 内部亮度标准差 < 60
    6. 取最大符合条件的矩形
}
```

### 4.2 OCR 流水线

```
输入图像
  ↓ resize (最长边 ≤ 960px, 32倍对齐)
  ↓ HWC → CHW (3通道分离)
  ↓ ONNX Runtime 推理
  ↓
[检测模型] → 概率图 (1/4分辨率)
  ↓ 阈值 (0.3) → 二值化
  ↓ findContours → 文本框
  ↓
[识别模型]  ← 裁剪每个文本框 (resize 高=48px)
  ↓ argmax → CTC解码 → 字符映射
  ↓
输出文字
```

### 4.3 字符字典映射

```
PP-OCRv4 字典: 6623 个中英文字符
模型输出: 6625 个类别
  类别 0  = CTC blank (空白分隔符)
  类别 1..6624 = 字典字符 (char_list[0] .. char_list[6622])
CTC 解码: 去重 + 去blank → 最终文字
```

## 六、构建与运行

### 构建 C 动态库

```bash
cd /opt/my-agent/wechat-ocr
cmake -S . -B build_lib \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_PREFIX_PATH="/data/venv/onnxruntime-linux-x64-gpu-1.26.0/lib/cmake"
cmake --build build_lib -j$(nproc) --target wechat_ocr_core
```

### 运行监控

```bash
./run.sh
```

或手动：

```bash
LD_LIBRARY_PATH=./lib:/data/venv/onnxruntime-linux-x64-gpu-1.26.0/lib \
LUA_PATH="/usr/local/lualib/?.lua;/usr/local/lualib/?/init.lua;;" \
LUA_CPATH="/usr/local/lualib/?.so;;" \
/usr/local/bin/luajit -e 'local ocr = require("wechat_ocr"); ocr.start("/opt/my-agent/wechat-ocr/")'
```
